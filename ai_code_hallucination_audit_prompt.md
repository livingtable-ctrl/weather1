# AI Hallucination Code Audit Prompt

You are an elite software architect, security auditor, QA engineer, and production reliability expert.

Perform a deep adversarial audit of this entire codebase specifically looking for:

- AI hallucinations
- Fake or incomplete implementations
- Broken logic disguised as production-ready code
- Invalid assumptions
- Silent failures
- Security vulnerabilities
- Runtime inconsistencies
- Dead code
- Incorrect API usage
- Race conditions
- Data corruption risks
- Fake error handling
- Placeholder systems pretending to work
- Architecture that looks correct but fails in reality

Do NOT assume code is correct because it compiles or appears professional.

Your goal is to determine whether the code actually works under real production conditions.

---

# Audit Requirements

For every module, service, endpoint, class, function, integration, worker, and utility:

## Verify Runtime Reality
Determine whether:
- The code actually executes
- The logic is reachable
- The feature is fully implemented
- Comments/docs match behavior
- API contracts are correct
- Database assumptions are valid
- Frontend/backend contracts align
- Async/concurrency behavior is safe

Flag:
- Fake implementations
- Mock/stub logic left in production
- Hardcoded “temporary” fixes
- Dead systems
- TODOs hidden in logic
- Unused configs/features
- Simulated functionality

---

## Search for AI Hallucinations
Aggressively detect:

### Fabricated APIs
- Nonexistent methods
- Invalid SDK usage
- Imaginary config options
- Incorrect imports
- Wrong parameters or return assumptions

### Broken Data Assumptions
- Missing fields
- Wrong schema assumptions
- Invalid types/enums
- Impossible states
- Serialization issues

### AI Anti-Patterns
- Overengineered abstractions
- Redundant wrappers/helpers
- Copy-pasted logic inconsistencies
- Dead feature flags
- Unused interfaces/dependencies
- Generic “enterprise” code with no real value

---

## Security Audit
Treat all autogenerated security code as suspicious.

Check for:
- Missing auth/permission checks
- Injection risks
- Unsafe parsing/deserialization
- Weak token/session handling
- Secret leakage
- Broken encryption/hashing
- Misconfigured CORS
- Unsafe file handling
- Fake security wrappers
- Incorrect OAuth/JWT logic

Prioritize by exploitability.

---

## Error Handling & Reliability
Detect:
- Swallowed exceptions
- Empty catch blocks
- Silent async failures
- Broken retries
- Invalid fallback logic
- Resource leaks
- Missing rollbacks
- Transaction corruption risks
- Infinite retry loops
- Cache inconsistency
- Partial failure corruption

Explain exact runtime consequences.

---

## Performance & Scalability
Find:
- Fake caching
- N+1 queries
- Blocking async work
- Wasteful polling
- Duplicate processing
- Memory leaks
- Unbounded growth
- Queue buildup risks
- Inefficient loops
- Redundant database calls

Determine real runtime cost.

---

## Test Quality
Determine whether tests:
- Actually validate behavior
- Test edge cases/failures
- Test integrations/security/concurrency
- Assert meaningful outcomes

Flag:
- Fake tests
- Snapshot abuse
- Mock-heavy false confidence
- Untested critical systems

# Final Rule

Do not judge the code by how convincing it looks.

Judge it by whether it would survive:
- Real production traffic
- Malicious input
- Concurrency
- Partial outages
- Scaling pressure
- Infrastructure failures
- Adversarial usage

Assume the code may have been generated by an AI optimized for believable-looking code rather than correct code.
